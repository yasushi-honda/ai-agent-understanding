1. そもそも「AIエージェント」って何？

AIエージェントとは、自分で考え、必要な道具を選び、実際に行動してくれるAIのことです。

ChatGPTのような「質問に答えるAI」と違い、エージェントは「調べる」「ファイルを開く」「メールを送る」といった実行まで担当します。人間がいちいち手順を指示しなくても、ゴールを伝えれば自分で段取りしてくれます。

たとえると…
新人アシスタントに「経費精算をお願い」と頼む感じです。アシスタントは「領収書を探す → システムに入力 → 承認者に送る」という段取りを自分で考えて動きます。AIエージェントは、この“考えて動く”部分をLLMとツールで実現したものです。

2. 脳になるLLM（大規模言語モデル）

LLMはエージェントの脳。言葉を理解し、何をすべきか判断する部分です。

LLM（Large Language Model）は、膨大な文章を学習したAIモデルです。文章の意味を読み取り、次にどう動くべきかを言葉で考えられます。

エージェントの中での役割

  • ユーザーの依頼を読み取る（意図の理解）
  • どの道具（Function/Tool）を使えばよいか選ぶ
  • 道具の実行結果を読んで、次の一手を決める
  • 最終的な答えを自然な日本語で返す

主なモデル（2026年4月時点）

提供元モデル名特徴

AnthropicClaude Sonnet 4.6速度と賢さのバランス型

イメージ：LLM単体はとても賢い“相談役”。ただし手足がないので、実際に何かを動かすには次章のFunction Callingが必要です。

3. 手足を動かすFunction Calling

Function Callingは、LLMが外部の機能を呼び出すための「注文票」の仕組みです。

LLMはそのままでは文章しか返せません。そこで「この関数を、この引数で呼んでください」というJSON形式の注文票を出せるようにしたのがFunction Callingです。注文票を受け取ったホスト（エージェントの土台）が、実際に関数を実行し、結果をLLMに戻します。

動きの流れ

  1. ユーザーが自然言語で依頼する
  2. LLMが「どの関数を呼ぶか」「引数は何か」を考える
  3. 関数名と引数をJSONで出力する（＝注文票）
  4. ホストが実際にその関数を実行する
  5. 結果をLLMに渡し、LLMがユーザーに回答する

注文票の例

{
  "name": "calc_total",
  "arguments": { "price": 1200, "quantity": 3 }
}
たとえると…レストランでの注文伝票です。お客さん（LLM）は「これ食べたい」と書くだけで、実際に料理するのは厨房（実行環境）。LLMは調理方法を知らなくてよい、という分業がポイントです。

4. 世界とつながるMCP

MCP（Model Context Protocol）は、AIと外部ツール・データを接続するための共通規格です。

Function Callingはとても便利ですが、ツールごとに呼び出し方を作り込む必要がありました。MCPは「AI ⇄ ツール」の間に共通のコンセントを用意し、どんなサービスでも同じ手順で接続できるようにしたものです。Anthropicが2024年に公開し、現在では多くのAIクライアントが対応しています。

MCPサーバーが提供する3つのもの

  • Tools：実行できる機能（例：メール送信、SQL実行）
  • Resources：読み取れるデータ（例：ドキュメント、ファイル）
  • Prompts：再利用できる指示テンプレート
たとえると…世界共通のUSB-Cコネクタ。以前はツールごとに違うケーブルが必要でしたが、MCPという共通規格ができたことで「差すだけで動く」世界になりました。

5. 全体像：LLM × MCP × Function Calling

3つの要素は、次のような順番で連携します。

  1. ユーザーが自然言語で依頼する
  2. LLMが「どの道具を使うか」を判断する
  3. LLMがFunction Callingの形式で注文票を出す
  4. MCPサーバーが注文票を受け取り、実際のツールを実行する
  5. 結果がLLMに戻り、LLMが自然な言葉で答える
覚え方：LLMは「脳」、Function Callingは「注文票」、MCPは「配膳係」、Toolsは「厨房」。

6. エージェントの代表的な動き方

エージェントにはいくつかの“思考スタイル”があります。代表的な4つを紹介します。

① ReAct（考える→動く→見る）

最も基本のループ。「Thought（思考）→ Action（行動）→ Observation（観察）」を繰り返します。Claude Codeもこの形を基本にしています。

② Plan & Execute（計画してから実行）

最初に大きな計画を立てて、順番に実行していくスタイル。複雑な作業や複数ステップのタスクに向きます。

③ Tool Use Agent（道具選択型）

使えるツール群の中から、状況に合わせて最適なものを選んで実行します。Claude CodeやChatGPTの「Tools」機能が該当します。

④ Reflection / Self-Critique（自己評価型）

自分の出力を別の視点で見直して改善します。Anthropicが推奨するGenerator-Evaluator分離もこの考え方の一つです。

たとえると…ReActは「動きながら考える人」、Plan & Executeは「先に計画書を書く人」、Tool Useは「道具箱から最適なものを選ぶ職人」、Reflectionは「自分の作品を見直す作家」です。

7. 複数のエージェントが協力する世界

一人のエージェントで足りないときは、複数のエージェントがチームを組んで作業します。これをマルチエージェントといいます。

代表的な協調パターン

  • Orchestrator-Worker：リーダーがタスクを分配し、複数ワーカーが並行作業（例：フロント担当・バックエンド担当・テスト担当）
  • Peer-to-Peer：対等な立場のエージェント同士でやり取り
  • Pipeline：工場のライン式にA→B→Cと順番に処理
マルチエージェントのコツ：同じファイルを同時に触らない／結果は必ず検証する。人間のチーム運営と同じで、役割分担とレビューが大事です。
